## 2. 总体翻译方针

### 2.1 总原则

- 优先保留“写信的人是谁”与“此刻的情绪状态”。
- 不把原文译得过分顺滑。
- 不主动替作者修辞。
- 不主动消除重复。
- 不主动解释诗句。
- 保留删改痕迹、页内接写、断裂感、迟到感。

### 2.2 语言质地

这部作品不是稳定、均匀、现代出版物式的 prose。
它应该保留以下质地：

- 少年写作者的犹疑、绕路、自我修正；
- 老师的节制、讲解欲、知识密度；
- 诗歌里时而稚拙、时而锋利的跳跃；
- 非线性结构造成的后知后觉。

### 2.3 不能做的事

- 不把“我”的信统一润成成熟作家口吻。
- 不把“老师”的信译成纯学术说明文。
- 不把诗译成押韵优先的现代歌词。
- 不把日文里故意保留的重复，改成同义改写。
- 不把删改稿痕迹当成排版错误删掉。

## 3. 叙述与语气圣经

## 3.1 “我”的来信

关键词：

- 少年感
- 敏感
- 自我意识强
- 轻微过度思考
- 经常借景、借典、借别人语言来搭桥

中文风格要求：

- 句子可以长，但不能太工整。
- 可以保留“我也不知道怎么说”“是不是”“好像”这类犹疑感。
- 不要过度书面化。
- 不要把情绪都翻译成明确结论。

禁止倾向：

- 不要译成成熟散文家。
- 不要译成纯口语网文。
- 不要故意做“文艺腔”升级。

## 3.2 “老师”的回信

关键词：

- 克制
- 温和
- 有引导性
- 博识
- 后期带有强烈自责与自我否定

中文风格要求：

- 句式比“我”更稳。
- 解释典故时允许更整齐一点。
- 但仍应保留通信感，不要变成讲义。
- 后期谈罪与原谅时，要有压抑的重量感，不能空泛。

禁止倾向：

- 不要译成导师训话。
- 不要译成司法报告腔。
- 不要译得比原文更圣人化。

## 3.3 诗中的第一人称变化

原文诗里会出现：

- `僕`
- `私`
- `おれ`

翻译时不能一律抹平。

建议：

- `僕`：默认译 `我`
- `私`：默认仍译 `我`
- `おれ`：在语势明显变粗、变硬、变露骨时，可译 `我` 但要用句势体现更硬的力度

说明：

中文里第一人称变化不像日语那么显眼，因此差异主要靠句势、词语硬度、节奏来补。

## 3.4 第二人称变化

原文中常见：

- `君`
- `あなた`
- `貴方`
- `お前`

建议基线：

- 在书信正文里，大多数情况下统一译为 `你`
- 在诗中若出现 `お前`，不要机械译成粗暴口吻，但要保留更直接、更贴近身体的压迫感
- `貴方` 若明显带书写性、抒情性，可仍译 `你`，由诗的气息承担差别

原则：

中文不要为了区分而硬塞“汝”“你呀”“你这家伙”之类不自然译法。

## 4. 书信与版式规则

## 4.1 标题与结构

- 章节标题沿用整理阅读版的结构，不回退到原始编号。
- 每个阅读单元内部保留原文页感。
- 补遗区必须显式标注为补遗，不可伪装成正常线性章节。

## 4.2 书信套语

推荐译法：

- `拝啓` -> `敬启`
- `敬具` -> `敬具`
- `P.S.` -> `附言`
- `追伸` -> `又及` 或 `附言`

说明：

- `敬启 / 敬具` 保留了信札感，也能传达略带旧式的礼貌。
- 如果最终成稿觉得 `敬具` 过硬，可在定稿阶段统一评估是否改为 `此致 敬礼`，但底稿阶段不建议先改。

## 4.3 破折与分隔

- 原文中的 `---` 如果在整理阅读版里已拆成单元，则按单元翻译。
- 若后续回看原稿时仍遇到同页接写，不增补解释，只在注释层说明。

## 4.4 删改痕迹

- `~~删除~~` 必须保留。
- 删除线里的内容也要译。
- 不得将删除线内容并入正文当作正式版本。

## 4.5 脚注与校注

- `^[...]` 保留为脚注。
- 校注与译注必须区分。

建议：

- 原有校注：保留原位
- 新增译注：统一写作 `[译注]`

## 4.6 夹注与页内批语

- 以 `> ` 出现的内容，默认保留块引用形态。
- 它可能是：
  - 页边回应
  - 同纸补记
  - 引文
  - 诗中嵌入的旁注

翻译时不擅自改成普通正文。

## 10. 诗歌翻译规则

## 10.1 优先级

诗歌翻译优先级建议如下：

1. 说话人的情绪与姿态
2. 反复句的回响
3. 意象链的稳定
4. 行与行之间的呼吸
5. 押韵

说明：

这部作品的诗更依赖重复、推进、变体、删改痕迹和意象照应，不应优先追求整齐押韵。

## 10.2 行分割

- 原则上保留原行分割。
- 不把多行合并成整段抒情散文。
- 若中文长度过长，可定稿时微调，但底稿阶段尽量贴原排。

## 10.3 反复句

- 同一句反复出现时，优先保持同一译法。
- 只有在原文本身有变体时，译文才跟着变。

## 10.4 粗糙感

- 某些诗句本来就略显稚拙、跳接或突兀。
- 不要把它们全都打磨成“完美现代汉诗”。
- 但也不要把本可译成诗的句子，落成解释性的白话说明。
- 诗译宁可稍生、稍险，也不要太顺、太满、太像散文转述。

## 10.5 人称与关系

- 同一首诗里 `私 / 僕 / おれ` 的切换，是情绪线索。
- 同一首诗里 `あなた / 貴方 / お前` 的切换，也是关系线索。
- 中文即便无法逐字区分，也要在句势上让差异存在。

## 11. 待核问题清单

这些问题不影响开始翻译，但在正式定稿前必须统一：

1. 书名最终是否保留《一人称》而不译。
2. `拝啓 / 敬具` 最终是否保留日式信札套语。
3. `ルバート`、`火星へランデヴー` 这类标题是否保留外来词感。
4. 经典作品与诗集名称是否存在更通行的简中译名。
5. `祖父 / 祖母 / 叔父` 是否要在亲属关系明朗后统一成外祖系称呼。
6. `{漢字|讀音}` 这类标记最终如何落地：
   - 正文只译义
   - 正文译义并加注原读
   - 注释版保留，阅读版不保留
7. `10-1` 是否在最终译本中保留“异文”标记，或另加编者注。

## 12. 翻译批次输入模板

后续每次喂给模型时，建议固定携带以下信息：

### 12.1 必带材料

- 本次要翻译的阅读单元正文
- 前 1 到 2 组通信摘要
- 本文件中的相关术语表
- 已确定译名表
- 若本次含诗，则附上该诗前后剧情摘要

### 12.2 固定指令模板

可直接复用如下模板：

```text
请将以下《一人称》文本从日文译为简体中文。

要求：
1. 严格区分“我”和“老师”的语气。
2. 保留书信体、删改痕迹、脚注、块引用。
3. 不擅自解释诗句。
4. 反复句尽量保持统一译法。
5. 若遇到已有术语表，请优先遵守术语表。
6. 输出分为两部分：
   A. 可读译文
   B. 译者备注：仅列出存在歧义、需统一或需核对通行译名之处
```

## 13. 当前结论

可以开始翻译，但应遵守以下最低约束：

- 以整理阅读版为源文本
- 以本文件为统一术语与风格基线
- 诗与信分开看待
- 保留不稳定、未完成、删改中的文本感

一句话概括：

**这部作品的翻译，不该追求“像翻译过的”，而该追求“像这些信原本就以中文写成，但仍保留其手稿与迟到感”。**
